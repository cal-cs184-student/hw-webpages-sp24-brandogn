Task 1

The core idea of triangle rasterization is to color all of the pixels in your view buffer that fall within the bounds of a triangle defined by 3 points.

In this section, I implemented a simple algorithm that iterates through all the pixels within the bounding box of the triangle, and colors them if they are inside the triangle. Initially I used a method involving three dot product edge functions, however I later switched to using barycentric coordinates because it used less memory and less computations (no need to set up the Vector2D structs or check for orientation).

Task 2

The supersampling algorithm that I implemented was a small modification of the original rasterization algorithm. Instead of just calculating if the center point of a given pixel (x,y)(x, y)(x,y)﻿, I calculated sss﻿ samples, where sss﻿ is the sample rate. For storing these extra samples, I made sure to resize the sample_buffer by multiplying it by sss﻿, and I simply stored the samples next to each other in the sample_buffer array. Then, when resolving the samples to buffer, I took the average of all the consecutive samples corresponding to each (x,y)(x, y)(x,y)﻿. Essentially, we are modifying the rasterization and framebuffer parts of the pipeline.

Super sampling helps reduce aliasing by essentially blurring the original triangle signal:

Task 5

The idea of pixel sampling is to take some coordinates (x,y)(x, y)(x,y)﻿ on the view port, and figure out what color should be there based on some data such as a mapped texture. For texture mapping, I implemented pixel sampling by finding the barycentric coordinates of some (x,y)(x,y)(x,y)﻿ inside the triangle, then using those weights to interpolate between the three uv coordinates corresponding to the three vertices. Then I used the new, interpolated uv coordinate to sample from the original texture.

There are two pixel sampling methods that I implemented in this section: nearest and bilinear. The nearest method simply takes some coordinates (x,y)(x,y)(x,y)﻿, and rounds them to the nearest integer value, and returns the color stored at the exact integer coordinates. Bilinear involves taking the four surrounding integer coordinates, and returns an interpolation of the four colors at those coordinates.

Typically, bilinear interpolation works by multiplying each point based on the vertical and horizontal proportional distances. Here’s a visual illustration of billinear interpolation.

Task 6

The idea of level sampling is to adjust the texture we are sampling based on how small and compact the texture is on the view port. The goal of this is to prevent aliasing by blurring the places where we find that the texture is being too compacted, while not blurring the places where the texture is not too compact. For texture mapping, I used the formula to find the level based on the derivatives of the uv cordinates at a given point in a triangle, and used that level to index into which mipmap texture we sample from.

Using L_ZERO, you essentially only use memory for just a single copy of the full scale resolution texture, and you only sample once per pixel, though as we see below, aliasing is very apparent. For L_NEAREST, we need to use more memory to store all of the levels of the texture, but we still only have to sample once per pixel, but there is significant improvements in anti-aliasing, especially in areas where the textures seem to be compacted. For L_LINEAR, our memory and speed are similar to the nearest method, but we instead have to sample twice per pixel, and interpolate between those two samples. There is improvement in anti-aliasing though, as the levels are more gradual (we don’t jump between levels suddenly, and the in-between areas look a lot less aliased).
