Overview

In this project, we implemented a real-time, physics-based simulation of a cloth. This includes modeling the cloth using mass points and springs, implementing the physics behind the mass points and springs, implementing collisions with external objects and itself, and writing shaders to render out the cloth. A lot of the challenges we encountered were related to C++ syntax and other programming bugs, which we solved by diligent debugging and referencing edstem, and other class resources.

One Specific Challenge: One important problem for part 2 was misunderstanding that external_accelerations were supposed to apply each acceleration to every single object, rather than apply the ith acceleration to the ith object. This ended up accessing undefined memory which outputted garbage values when trying to index past the length of external_accelerations.

We included a variety of interesting visual bugs in the sections below for your reference.

Part 2: Simulation via Numerical Integration

Below are some images to show the basic functionality of our simulation which involved various physics equations and simulation techniques.

Varying Spring Constants ks

When the spring constant ks is very low, (5) the cloth acts as though it is very thin, sagging a lot and deforming quite significantly around the top. When ks is very high (10000), the cloth appears much stiffer, not deforming very much at the top of the bottom. The speed at which the cloth falls does not change very much.

Varying Densities

Density behaves in the opposite way that the spring constant behaves. When density is high, the cloth deforms more and almost appears thinner, whereas when density is low, the cloth deforms less and appears stiffer. Neither of these affect the rate at which it falls.

Varying Dampening

Damping affects the speed at which the cloth falls, as well as the number of ripples that form in the cloth as it approaches its rest state, however the final resting state of the cloth is the same regardless of which damping value we choose.

Part 5: Shading

A shader program is a program that takes in mesh data and outputs values to the surface to render out. In this case, we have a vertex shader takes in vertex data and outputs important data such as the normal, position, tangent, and uv coordinates after maybe doing some processing or transformations. The fragment shader then takes in that data and interpolates and processes the data across each polygon and outputs the color which should be rendered for the appropriate pixels on the screen. Below are some examples of applications of shader programs.

Blinn-Phong

At an abstract view, Blinn-Phong shading is a shader that builds on the Diffuse shader, which colors the surfaces of a mesh based on the surface normal. Phong shading adds a ambient and specular aspect to this, using the formula:

where kak_aka​﻿ is the constant for ambient lighting, kdk_dkd​﻿ is the constant for diffuse lighting, and ksk_sks​﻿ is the constant for specular lighting.

Below, we can see the different components, as well as the result with all of them combined. Ambient is simply an overall iridescence applied to all surfaces based on the ambient constant, diffuse is the base shader, and specular is the shiny aspects of the shader.

Bump Mapping / Displacement Mapping

Bump mapping involves taking the original Phong shading, and adjusting the normals of each shaded point based on some height texture that is uv mapped onto the surface. Displacement on the other hand does this and takes the next step of actually moving the points of the surface based on the height texture.

Below, we can see how both images use the same position and mapped brick texture, but the displacement has bumps in the cloth to reflect where height differences are apparent. The bump map preserves the shape, but still applies the lighting texture.

Variable Coarseness:

With a higher mesh coarseness, the displacement shaders seemed to be a lot more accurate in that there seemed to be a higher resolution for the parts of the mesh that were raised. With a mesh coarseness of 16, the surface of the sphere is rather jagged instead of following the brick pattern that is present, while with a mesh coarseness of 128, the surface much more resembles the brick pattern. The Bump shading didn’t seem to see any effects from adjusting the mesh coarseness, likely because it’s not displacing any parts of the surface.

Custom

We combined the phong shading with texture mapping and mirror shading to create an interesting result of a metallic ballincat43 colored cloth! This essentially involved multiplying the reflection shader, texture shader, and phong shader together. We tried addition, and found that adding the reflection results in a laminated look, but adding the texture created a faded look, so we opted to multiplying them all together. We also had to adjust the constant for the diffuse shading since it was making the texture dark.
